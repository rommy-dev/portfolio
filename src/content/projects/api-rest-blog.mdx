## Contexte du projet

Ce projet visait a fournir une API claire et modulaire pour alimenter un blog moderne. L'objectif etait d'avoir une base solide, documentee et securisee.

---

## Fonctionnalites principales

- **Authentification JWT** et gestion des roles
- **CRUD** des articles, categories et commentaires
- **Documentation** interactive de l'API
- **Pagination** et filtres sur les listes

---

## Architecture technique

### Backend — Node.js + Express

- Structure MVC legere
- Middlewares pour validation et securite
- Services pour isoler la logique metier

### Base de donnees — PostgreSQL

- Schema relationnel simple et evolutif
- Indexation des champs critiques

---

## Defis techniques rencontres

- **Securisation des endpoints** sensibles
- **Validation stricte** des payloads
- **Performance** sur les requetes listees

---

## Ce que j'ai appris

- Organiser une API maintenable
- Documenter correctement des endpoints
- Mettre en place une auth propre et testable

---

## Statut actuel

Le projet est **livre** et peut etre integre a n'importe quel front web ou mobile.
